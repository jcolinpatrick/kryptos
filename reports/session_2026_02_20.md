## 4. Non-Standard Cipher Structures (Explorer)

**E-EXPLORER-04/05/06:** Tested 9 non-standard hypotheses:

| Hypothesis | Best Score | Verdict |
|-----------|-----------|---------|
| Polybius-variant lookup tables | 4/24 | NOISE |
| Position-dependent alphabet switching (N=2..7) | 4/24 | NOISE |
| Dual-system interleave (N=2..5, periodic keys) | 24/24 at N=4,p=7 | **FALSE POSITIVE** |
| K1-K3 plaintext as running key | 5/24 | NOISE |
| Berlin Clock reading orders | 4/24 | NOISE |
| K2 coordinate-derived keys | 3/24 | NOISE |

| Misspelling-derived keys (CQUAE/EQUAL) | 4/24 | NOISE |

**Key finding:** The 24/24 interleave result was a false positive caused by underdetermination. With 4-way interleave at period 7, every residue class has at most 1 crib constraint, making 24/24 automatic. Monte Carlo confirmed: 50% of random configs hit 24/24.

**New eliminations:**
- 2-way and 3-way interleave with periodic sub-keys (period <= 7): ZERO configs at 24/24
- K1-K3 plaintext specifically eliminated as running key source (max 5/24)

**Bonus:** Polynomial key extension (degree 1-2): best fit 7/24. No clean algebraic function generates the known keystream.

**E-EXPLORER-07:** Theoretical analysis of K5's "shared coded words at same positions" constraint.

Key findings:

2. **Transposition not ruled out** -- same fixed permutation applies to both K4 and K5
3. **Key design converges on lookup table** -- all constraints (non-periodic, non-polynomial, position-dependent, reusable, hand-executable) point to coding charts
4. **Vigenere tableau as running key: NOISE** (5/24) -- 5 reading orders tested
5. **K5 CT would be decisive** -- depth-of-two analysis (CT4[i]-CT5[i] = PT4[i]-PT5[i] mod 26) would eliminate the key entirely

**Testable prediction:** K4's known keystream at crib positions should decrypt K5 CT to English words when K5 CT becomes available.
