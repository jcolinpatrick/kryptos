## 5. Result-Capture Gaps

### Gap 1: 208 scripts with results in reports only (HIGH severity)

The **single largest gap**. 56% of all experiments have results documented only in:
- Session reports (`archive/session_reports/*.md`, 27 reports)
- Progress archives (`progress_archive.md` at 107K chars, `PROGRESS.md` at 110K chars)
- Memory files (`memory/MEMORY.md`, `memory/eliminations.md`)

These results are **not machine-readable**. They exist as prose in markdown, not structured JSON. Consequences:
- The site builder cannot automatically ingest them (only 33 scripts have `overrides.toml` entries)
- No automated regression testing against prior results
- Cross-referencing requires human or LLM interpretation

### Gap 2: Novelty ledger disconnected from execution (MEDIUM severity)

The novelty system (`db/novelty_ledger.sqlite`) contains 144 hypotheses (71 proposed, 68 eliminated, 5 triaged) but all four execution-tracking tables (`runs`, `results`, `eliminations`, `checkpoints`) are **empty**. Experiments were executed through standalone scripts instead of the novelty pipeline. The system was designed but never used for execution tracking.

## 6. Trust Assessment

### Verdict: PARTIAL COVERAGE, GOOD ELIMINATION CONFIDENCE, POOR RESULT CAPTURE

#### What's trustworthy

- **The elimination landscape is comprehensive and internally consistent.** `docs/elimination_tiers.md` (247 lines), `memory/eliminations.md` (380+ lines), and `reports/final_synthesis.md` paint the same picture across 370+ experiments. Cross-verification by multiple agents (QA, Validator, FRAC) found zero contradictions.
- **The 72 scripts with JSON result files** have complete, non-zero, well-structured outputs. Every single one contains a clear verdict. All files in `results/` are intact.
- **The 208 scripts documented in reports** have verdicts recorded in prose. While not machine-readable, they are consistent with each other and with the elimination tiers.
- **Bean impossibility (E-FRAC-35)** and the multi-objective oracle retroactively validate all prior NOISE results at eliminated periods. This is a mathematical proof, not an empirical result — it holds regardless of whether any particular script ran.
- **All 513 unit tests pass.** The scoring oracle (`score_candidate()`) self-verifies at import time. The kernel is sound.

#### What's uncertain

- **~73 scripts (20%)** have no result file AND no documentation reference. Their verdicts in the framework inventory were inferred from code analysis, not execution evidence. These scripts probably ran during interactive sessions — they were committed alongside scripts that definitely ran — but there is no proof.
- **3 pending job scripts** (ant_002, ant_003, ant_004) definitely never ran. `ant_003` tests a unique hypothesis (merged K3+K4 block) worth executing.

#### What's poor

- **Result capture is the weakest link.** Only 19% of scripts have structured (JSON) result files. The other 81% either have results buried in markdown prose or have no captured output at all.
- **The novelty ledger infrastructure was built but largely unused.** All execution went through standalone scripts, bypassing the `runs/results/eliminations` tracking tables. 71 proposed hypotheses remain untested in the ledger.
- **No centralized experiment registry** maps every script to its execution status, result file, and verdict in one machine-readable place. `framework_inventory.json` was the closest attempt but was generated post-hoc by an agent reading source code; its "unknown" verdicts are unreliable.

#### Quantified trust

| Metric | Value |
|---|---|
| Scripts with machine-readable results | 72 / 373 (19%) |
| Scripts with any documented verdict | 280 / 373 (75%) |
| Scripts with no evidence of execution | ~73 / 373 (20%) |
| Scripts confirmed never run | 3 (in `jobs/pending/`) |
| Elimination claims with repro commands | 33 (via `overrides.toml`) |
| Elimination claims in docs without repro commands | ~80+ |
| False positive rate in results | 0% (all claimed signals were identified as false positives) |

### Bottom line

The project has **high confidence in its cryptographic eliminations** — the mathematical conclusions are sound and internally consistent. It has **low confidence in its execution bookkeeping** — many scripts lack proof of having been run, and most results exist only in prose. The 3 genuinely unexecuted job scripts represent the only concrete coverage gap in the hypothesis space. The ~73 uncertain scripts are very likely duplicative of work documented elsewhere under different experiment IDs or in session report prose.

**Highest-leverage actions:**
1. Run `ant_003_merged_transposition.py` (unique untested hypothesis)
2. Re-run 3 scripts to capture missing result files
3. Move 5 completed `ad-h*` jobs from `pending/` to `done/`
4. Fix 4 documentation errors in crypto field manual and ground truth docs
